# Initializing the LucraClient

Import from `lucra-web-sdk`. The client enforces a singleton - only one instance can exist at a time.

**Initialize once at app startup:**

```typescript
import { LucraClient } from 'lucra-web-sdk';

LucraClient.initialize({
  apiKey: "<your Lucra api key>",
  tenantId: "<your Lucra tenant id>",
  env: "sandbox",
});
```

**Retrieve the instance anywhere:**

```typescript
import { LucraClient } from 'lucra-web-sdk';

const client = LucraClient.getInstance();
```

Calling `initialize` a second time throws. Calling `getInstance` before `initialize` throws.

**Cleanup (e.g. on logout):**

```typescript
LucraClient.destroy();
```

After `destroy`, `initialize` can be called again.

**React example:**

```typescript
// app entry point (e.g. main.tsx)
import { LucraClient } from 'lucra-web-sdk';

LucraClient.initialize({
  apiKey: "<your Lucra api key>",
  tenantId: "<your Lucra tenant id>",
  env: "production",
});
```

```typescript
// any component
import { LucraClient } from 'lucra-web-sdk';

function MyComponent() {
  const client = LucraClient.getInstance();
  // ...
}
```

## Awaiting Initialization

The client exposes `ready` (a Promise), `isInitialized` (a boolean), and `user` (the current Lucra user, or `null`) so consumers can know when the iframe has finished initializing.

```typescript
const client = LucraClient.getInstance();
client.open(container).home();

await client.ready;
const user = client.user;
```

You can also subscribe to the `initialized` event for the same signal. `client.ready` rejects if initialization fails.
