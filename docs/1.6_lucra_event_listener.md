# Lucra Event Listener

## v1: .on() / .off()

Subscribe to events from Lucra using `.on()`. The listener receives a typed `data` payload specific to each event.

```typescript
import { LucraClient } from 'lucra-web-sdk';

const client = LucraClient.getInstance();

client.on('loginSuccess', (user) => {
  console.log('User logged in', user);
});

client.on('matchupCreated', ({ matchupId }) => {
  console.log('Matchup created:', matchupId);
});
```

To stop listening, pass the same function reference to `.off()`:

```typescript
const handleUserInfo = (user) => { /* ... */ };

client.on('userInfo', handleUserInfo);
// later...
client.off('userInfo', handleUserInfo);
```

### Available events

| Event | Payload | Description |
| --- | --- | --- |
| `loginSuccess` | `LucraLoginSuccessBody` | User logged into Lucra |
| `userInfo` | `LucraUserInfoBody` | User data updated |
| `matchupCreated` | `{ matchupId: string }` | User created a matchup |
| `matchupStarted` | `{ matchupId: string }` | Matchup owner started the matchup |
| `matchupAccepted` | `{ matchupId: string }` | User joined someone else's matchup |
| `matchupCanceled` | `{ matchupId: string }` | User canceled a matchup |
| `activeMatchupStarted` | `{ matchupId: string }` | An active matchup started |
| `tournamentJoined` | `{ matchupId: string }` | User joined a tournament |
| `convertToCredit` | `{ amount: number }` | User requested a credit withdrawal |
| `deepLink` | `{ url: string }` | Lucra is requesting a URL to open in your app |
| `navigationEvent` | `{ url: string, page?: LucraPage }` | User navigated within Lucra |
| `claimReward` | `{ reward: LucraReward }` | User is redeeming a Free To Play reward |
| `exitLucra` | `void` | User attempted to exit Lucra |
| `kycComplete` | `void` | User finished the KYC flow |
| `demographicComplete` | `void` | User finished the demographic collection flow |
| `initialized` | `{ success: boolean }` | Lucra iframe finished initializing |

`kycComplete` fires when the user finishes the KYC flow opened via `kyc()` (see [Lucra Flows](1.3_lucraflows.md)). Lucra does not navigate away on completion, so the embedding app decides what happens next — typically closing the dialog it opened KYC in:

```typescript
const dialog = client.dialog().kyc();

client.on('kycComplete', () => {
  dialog.close();
});
```

`demographicComplete` fires when the user finishes the demographic collection flow opened via `demographic()` (see [Lucra Flows](1.3_lucraflows.md)). As with KYC, Lucra does not navigate away on completion, so the embedding app closes the dialog it opened:

```typescript
const dialog = client.dialog().demographic();

client.on('demographicComplete', () => {
  dialog.close();
});
```

For matchup invite URL generation, see [Deeplinks](1.4_deeplinks.md).

### Messages you can send to LucraClient

`userUpdated` - update the LucraClient user with known information about the current user. This information will be persisted to LucraSports and will be used to pre-fill forms the user will have to fill out for things like Identity Verification.

You may optionally include a `metadata` property as valid JSON related to the Lucra user. Key/value pairs should be of type `String` only. _Note_: On the `metadata` object, the `externalId` property will be used to index, if provided. Passing in metadata in this way will be used to link client users with the Lucra user for autosettlement identification purposes.

`navigate` - navigate the Lucra iframe to a specific pathname within the Lucra app.

```typescript
client.sendMessage.navigate({ pathname: '/app/home' });
```

### Async API

The `api` object exposes Promise-based requests to Lucra.

`api.achievements()` resolves with the current user's achievements.

```typescript
const { achievements } = await client.api.achievements();
```

`api.tournaments()` resolves with the list of recommended pool tournaments.

```typescript
const { tournaments } = await client.api.tournaments();
```

Each `LucraTournament` has `matchupId`, `canJoinTournament`, and a nested `matchup` object with `starts_at`, `expires_at`, `pool_tournament_details` (`title`, `buy_in_amount`, `icon_url`, `game_id`, `type`), `pool_payout_reward_structures`, and `pool_tournament_leaderboard_aggregate`.

`api.tournament(matchupId)` resolves with a single tournament's full detail.

```typescript
const { tournament } = await client.api.tournament(matchupId);
```

`tournament` is a `LucraTournamentDetail | null` with fields such as `title`, `buy_in_amount`, `payout_structure`, `total_participants`, `leaderboard`, and `user_leaderboard_row`.

`api.tournamentLeaderboard(matchupId, pagination?)` resolves with one leaderboard page. Pass `{ limit, offset }` to paginate, then merge pages client-side.

```typescript
const { ui_tournament_details } = await client.api.tournamentLeaderboard(matchupId, {
  limit: 20,
  offset: 0,
});
const rows = ui_tournament_details?.leaderboard?.rows ?? [];
```

`api.joinTournament(tournamentId)` requests joining the tournament with the given id and resolves with the joined `matchupId`.

```typescript
const { matchupId } = await client.api.joinTournament(tournamentId);
```

Call after `initialized` has fired (or `await client.ready`). The request times out after 15 seconds.

#### Handling join errors

When a join is blocked, the promise rejects with a `LucraApiError` whose `code` (a `LucraApiErrorCode`) tells you why, so your app can react — for example by routing the user to verification, adding funds, completing their profile, or showing a location message. Catch it and switch on `code`:

```typescript
import { LucraApiError, LucraApiErrorCode } from 'lucra-web-sdk';

try {
  const { matchupId } = await client.api.joinTournament(tournamentId);
} catch (e) {
  if (e instanceof LucraApiError) {
    switch (e.code) {
      case LucraApiErrorCode.unverified:
        // user has not completed identity verification
        break;
      case LucraApiErrorCode.insufficientFunds:
        // balance does not cover the buy-in
        break;
      case LucraApiErrorCode.demographicInformationMissing:
        // required profile/demographic info is missing
        break;
      case LucraApiErrorCode.locationError:
        // user's location could not be verified
        break;
    }
  }
}
```

| `LucraApiErrorCode` | Value | Meaning |
| --- | --- | --- |
| `unverified` | `UNVERIFIED` | The user has not completed identity verification |
| `insufficientFunds` | `INSUFFICIENT_FUNDS` | The user's balance does not cover the buy-in |
| `demographicInformationMissing` | `DEMOGRAPHIC_INFORMATION_MISSING` | Required profile/demographic information is missing |
| `locationError` | `LOCATION_ERROR` | The user's location could not be verified |

`LucraApiError` extends `Error` (so `message` is populated) and carries `code`. These codes are not specific to `joinTournament` — other API methods may reject with `LucraApiError` in the future. Other rejections (the 15s timeout, or a request superseded by a newer call) reject with a plain string, not a `LucraApiError`, so the `instanceof` check distinguishes them.
