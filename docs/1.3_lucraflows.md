# Lucra Flows Setup and Usage

### Open Lucra

```typescript
import { LucraClient } from 'lucra-web-sdk';

const client = LucraClient.getInstance();
client.open(element, phoneNumber?)
  // and one of the below
  .profile()
  .wallet()
  .home(locationId?)
  .deposit()
  .withdraw()
  .createMatchup(gameId?)
  .matchupDetails(matchupId)
  .tournamentDetails(matchupId)
  .deepLink(url)
  .kyc()
  .demographic()
```

Where phone number (if valid) will automatically send the SMS verification code, skipping the step where the user must enter phone number manually.

### Visibility and Placement

Hide the Lucra experience without removing it from the DOM:

```typescript
const client = LucraClient.getInstance();
client.hide();
```

Restore it:

```typescript
client.show();
```

Move the iframe to a different container element. Re-parenting an iframe causes the browser to reload its contents, so the Lucra page reloads after a `moveTo`. To present a route as an overlay without reloading, use a [dialog](#dialogs) instead.

```typescript
client.moveTo(otherElement);
```

Start the experience hidden from the very first render:

```typescript
client.open(element, phoneNumber, { hidden: true }).home();
// later...
client.show();
```

### Dialogs

Open any Lucra route as a fullscreen dialog. Unlike `moveTo`, a dialog does not re-parent the iframe, so the Lucra page is not reloaded — it navigates in place and keeps its session.

```typescript
import { LucraClient } from 'lucra-web-sdk';

const client = LucraClient.getInstance();
client.dialog()
  // and one of the below
  .profile()
  .wallet()
  .home(locationId?)
  .deposit()
  .withdraw()
  .createMatchup(gameId?)
  .matchupDetails(matchupId)
  .tournamentDetails(matchupId)
  .deepLink(url)
  .kyc()
  .demographic()
```

When KYC is opened as a dialog, Lucra does not navigate away once the user finishes — it emits a [`kycComplete`](1.6_lucra_event_listener.md) event so the embedding app can close the dialog:

```typescript
const dialog = client.dialog().kyc();
client.on('kycComplete', () => dialog.close());
```

Demographic collection works the same way — opened as a dialog it emits a [`demographicComplete`](1.6_lucra_event_listener.md) event so the embedding app can close the dialog:

```typescript
const dialog = client.dialog().demographic();
client.on('demographicComplete', () => dialog.close());
```

The element the iframe currently lives in (the one passed to `open()` / `moveTo()`) is styled as a fullscreen overlay while open, and restored on close.

Each route returns a `LucraDialog` handle:

```typescript
const dialog = client.dialog().wallet();

dialog.onClose(() => {
  // fires when the dialog closes (in-app close, Escape, or dialog.close())
});

dialog.close(); // close it programmatically
```

A dialog closes when the user taps the Lucra page's close control, presses `Escape`, or you call `dialog.close()`. `Escape` is observed on the parent page only — because the Lucra page renders in a cross-origin iframe, key presses while focus is inside it are not visible to the parent, so the page's own close control is the focus-independent way to dismiss. While a dialog is open the in-app close is handled by the dialog itself, so your `exitLucra` listener does not fire for it — see [Exiting Lucra](1.5_exiting_lucra.md).

### Popups

Open the deposit flow in a real browser popup window. Unlike `open()`, `redirect()`, and `dialog()` — which all render Lucra inside an iframe — a popup is a top-level browsing context. This is required for the deposit flow because Apple Pay does not run inside a cross-origin iframe. The popup flow is scoped to deposit only.

```typescript
import { LucraClient } from 'lucra-web-sdk';

const client = LucraClient.getInstance();
const popup = client.popup().deposit();
```

`popup().deposit()` must be called from a user gesture (e.g. directly inside a click handler), otherwise the browser blocks the popup and the call throws. It does not require `open()` first — the popup is standalone.

It returns a `LucraPopup` handle:

```typescript
const popup = client.popup().deposit();

popup.onClose((result) => {
  // Fires when the popup closes. `result` is { toastType, message } when the
  // deposit reported an outcome before closing, or undefined if the user
  // dismissed the popup without completing it.
  if (result?.toastType === "success") {
    // refresh balance, show confirmation, etc.
  }
});

popup.close(); // close it programmatically
```

A popup closes when the user closes the window, when the deposit completes (the Lucra page closes itself), or when you call `popup.close()`.
